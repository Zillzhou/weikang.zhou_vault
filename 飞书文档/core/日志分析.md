# 日志分析

## 日志分析
RDSCore 基于 Log分析器 分析调度日志，在编辑日志分析文档时，应遵循下述原则：
日志分析文档以问题类型作为分类依据，建立子文档；问题类型分类依据应考虑：具有 相似的观测指标、相似的处理流程、相似的功能模块、相似的问题表现
如：充电问题日志都会看控制权、电量、任务状态等指标；G-MAPF 问题日志都会看参数、下发路线等指标
### 在子文档中应使用分级标题管理不同问题，便于查看与索引
日志分析上手不易，应做到由浅入深，循序渐进，尽可能详细附带过程截图
在 Coding 处理问题如具有典型性，且分析问题具有明显的可归纳逻辑，应输出为日志分析文档
### 日常开发排查问题，如流程较复杂，亦可记录于此，避免遗忘
建议与 FAQ 进行关联，形成 发现问题-->分析问题-->解决问题 的流程
