## 运单状态说明
## None ：空状态
TOBEDISPATCHED ：订单尚未被分配给小车。
CREATED ：运单已经分配给机器人但是还未被执行。
RUNNING ：运单正在被机器人执行中。
WAITING ：运单中所有动作块都已经被执行完成了，但是运单没有被封口，等待后续指令。
FINISHED ：运单完成（终态）。
FAILED ：运单在执行过程中任务失败，可能是机器人执行任务失败导致的。
STOPPED ：运单在非正常状态下完成，如被人为终止时（终态）。

## 动作块的状态说明
### 0.1.8.220813 版本之前
## None ：空状态
### CREATED ：动作块在运单中创建，还未执行
## RUNNING ：动作块被执行中
### WAITING ：动作块已经被完成，但是出于延迟完成的状态
## SUSPENDED ： 动作块被暂停
FINISHED ：动作块完成（终态）。
MANUAL_FINISHED : 动作块被人工完成 （终态）。
FAILED ：动作块在执行过程中出错。
STOPPED ：动作块被人为终止（终态）。
### 0.1.8.220822 版本之后
## POSTING : 正在执行动作块回调
### MANUALLY_FINISHED : 动作块被手动完成
### CREATED : 动作块已被创建，还未执行
### RUNNING : 正在执行动作块中的机器人动作
WAITING : 机器人动作和回调已完成，在等待拼单或者放行信号
### SUSPENDED : 动作块的机器人动作被暂停
### FINISHED : 动作块完成（终态）
## NONE : 空状态
### STOPPED : 动作块被终止（终态）
