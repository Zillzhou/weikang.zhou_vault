# 赶车（autoMovablePark）

## 赶车（autoMovablePark）
有任务的机器人被无任务的机器人阻挡时，需移动无任务的机器人到一个不会阻挡的位置。
core 的处理策略是将阻挡的车调度到其绑定的、能够到达的最近的 PP 点。
此处无任务是指机器人当前没有执行运单或正执行自动停靠任务，阻挡分为两种情况，一种是处于其他机器人将要行驶的线路上，另一种不在其他机器人行驶的路线上，但若其他机器人继续行驶会与其相撞。

## 注意
任何车被机器人阻挡了，如果推车功能打开，机器人满足被驱赶条件，就可以触发赶车。例如：A被B阻挡，B被驱赶，当被驱赶的 B 车被 C 阻挡，也会驱赶 C。
仅可驱赶无任务的车，如果车正在 RUNNING 或处于 WAITING 状态，不可被驱赶
## 被驱赶的车必须有可达的 PP 点
