# 可接单状态

## 可接单状态
## @丁禹伯
机器人在的可接单状态有三个，一个是 可接单 ， 一个是 不接单且占用资源 ，一个是 不接单不占用资源 。
机器人处于可接单状态，是机器人可以参与下发运单的分配的必要条件，除此之外，服务器需要有机器人的控制权，机器人的定位已经被确认，机器人的电量正常，机器人正在使用的地图已添加到场景中。
机器人如果属于不接单且占用资源状态，那么机器人不接收新的任务。但是，机器人还是在场景中占用资源，因此，其他车可能会被当前车挡住。如果机器人有故障，我们可以将机器人设置为不接单且占用资源。
机器人如果属于不接单不占用资源，那么机器人不接收新的任务，并且机器人不在场景中占用资源，车子不会被当前车阻挡。如果机器人有故障，并且被人推离了线路，我们可以将机器人设置为不接单且不占用资源。
当调度重启时，并且存在失联且占用资源的机器人，那么调度会报错 52113 ，并且不会执行任何运单。因为，调度系统不知道失联机器人正确位置，所以让任何机器人移动都可能产生危险。此时，我们可以将失联的机器人，推离线路，并且设置为不占用资源，调度就可以正常运行了。

## Error52113
## 提示：
Unignore robot is disconnet.
## 触发条件：
### Core 重启时，存在失联的机器人占用资源
## 解决方式：
将失联机器人设置为不接单不占用资源。或者将失联机器人上线。
