## 如何正确的更新脚本文件
当项目已经实施过，任务链、库位、天风任务等已经配置完毕，现场已经正常运行。此时由于新需求或者现场出现问题，需要更新脚本文件时，该如何正确操作，能保证最小的变动和工作量，请务必按以下步骤一步步操作。
## 从Roboshop云端更新
### 从Roboshop云端更新脚本操作如下：
从coding或者研发获取项目唯一码，
### 打开Roboshop脚本界面，点击拉取按钮
## 输入项目唯一码，获取记录
## 点击【下载】，选项全部选否，确认
## 将更新后的文件【推送】
## 运行一遍新上传的脚本
点击【刷新】，【全部拉取】，查看 json 文件，按需求修改需要修改的参数的 value 值

## 从工单下载文件更新
### 如果是从coding附件里下载的脚本文件，按以下步骤更新：
查看项目当前正在使用的脚本文件，并记住文件名，如 containerRobot.py 
鼠标右键【复制并添加】，将复制的文件作为备份文件，然后删除原文件
### 从 coding 或者相关研发人员处获得新的脚本文件
如果新脚本文件名和之前不一样，需要将新的脚本文件名修改为和之前一样，如：修改 containerRobot-20240416.py 为 containerRobot.py
### 将新脚本文件上传至 Roboshop【脚本文件】处
## 运行一遍新上传的脚本
点击【刷新】，【全部拉取】，查看 json 文件，按需求修改需要修改的参数的 value 值
