# SyncService 页面说明

## SyncService 页面说明
## 一、介绍
SyncService 是一款基于在双机热备环境下的数据同步工具，可以帮助用户在多个设备之间同步。它支持跨平台，可以在 Linux 等操作系统上运行，并提供了 Web 界面和命令行界面两种操作方式。
SyncService 的 Web 界面提供了丰富的功能和选项，包括文件夹管理和设备管理等。用户可以通过 Web 界面添加、编辑、删除文件夹，设置同步选项，管理设备，查看同步状态和日志等。
在文件夹管理界面，用户可以添加新的文件夹，并指定同步的路径、同步设备等选项。用户还可以设置忽略规则，指定哪些文件或文件夹不需要同步，以及设置版本控制选项。
在设备管理界面，用户可以添加、编辑、删除设备，并设置同步选项、访问权限等。用户还可以查看设备的同步状态、连接状态和版本信息等。
总之 SyncService 是一款功能强大、易于使用的数据同步工具，可以帮助用户轻松地在多个设备之间同步文件和文件夹。它提供了丰富的功能和选项，完全可以满足个人和企业的不同需求。

### 除了不启用版本控制外，选项中另外 4 种版本控制含义如下：
回收站式版本控制是指将删除的文件先放入回收站，以便用户在需要时可以恢复这些文件。这种版本控制方式可以避免用户误删文件导致数据丢失的情况。
简易版本控制是指系统只会保存文件的最新版本，而不会保存历史版本。这种版本控制方式适用于对文件变化历史不敏感的场景，例如临时文件或日志文件等。
阶段版本控制是指系统会将文件的变化分成多个阶段，每个阶段对应一个版本。这种版本控制方式可以让用户更加清晰地了解文件的变化历史，以便更好地管理文件。
外部版本控制是指将文件的版本控制交给外部工具来管理，例如Git等工具。这种版本控制方式可以让用户更加灵活地管理文件的版本控制，同时可以与其他工具进行集成。
需要注意的是，启用版本控制会占用一定的磁盘空间。因此，如果存储空间有限，可以根据需要不启用版本控制。

## 监视更改
勾选是否开启【监视更改】，以便文件在发生更改时能立即被检测到。这种方法可以更快地检测到更改，并且不会占用太多系统资源。
## 文件夹类型
可供选择的文件夹类型有：发送与接收、仅发送、仅接收和加密接收。根据文件夹类型选择的不同，后续的内容也会有对应的变化。
a. 发送与接收：双向同步的文件夹类型，本地和远程设备之间的文件同步是相互的，支持文件的增量同步和版本控制。
b. 仅发送：只有本地设备向远程设备发送文件的文件夹类型，本地设备可以修改文件并将其发送到远程设备，但无法接收来自远程设备的文件。
c. 仅接收：只有远程设备向本地设备发送文件的文件夹类型，本地设备无法修改文件，只能接收来自远程设备的文件。
d.加密接收：只有远程设备向本地设备发送加密文件的文件夹类型，本地设备无法修改加密文件，只能接收来自远程设备的加密文件。该文件夹类型需要在本地设备和远程设备之间进行密钥交换，以确保文件的安全性和保密性。
需要注意的是，不同的文件夹类型适用于不同的场景和用途。用户可以根据自己的需求选择适当的文件夹类型，以实现更加灵活和高效的文件同步。
## 最低可用磁盘空间
默认 1% ，用户可根据需要自行修改。
## 所有权
a. 同步所有权：如果选择了同步所有权选项，当用户同步文件夹时，系统会尝试将文件的所有权信息（例如文件的所有者、组、权限等）与远程设备同步。这意味着，如果远程设备没有相应的所有权信息，系统会尝试将本地设备的所有权信息应用到文件上。这种方式可以确保文件在不同设备之间的所有权信息相同。
b. 发送所有权：如果选择了发送所有权选项，当用户同步文件夹时，系统只会将文件发送到远程设备，而不会尝试将本地设备的所有权信息应用到文件上。这意味着，如果远程设备没有相应的所有权信息，文件的所有权信息可能会发生变化。
需要注意的是，这些所有权设置只在文件同步时起作用，并不会影响文件本身的权限和所有者。如果用户需要更精细地控制文件的权限和所有者，可以使用操作系统提供的相应工具进行设置。
## 完整扫描时间
完整扫描间隔是指系统定期进行完整扫描的时间间隔。完整扫描是指系统会定期检查文件夹中的所有文件，以确保文件的一致性和完整性。在完整扫描期间，系统会比较本地文件和远程设备的文件，以检测是否存在任何差异或冲突。
完整扫描间隔的设置可以影响系统的性能和同步效率。如果完整扫描间隔设置得太短，系统会频繁进行完整扫描，导致性能下降和网络带宽浪费。如果完整扫描间隔设置得太长，系统可能会错过一些文件变化，导致同步不及时或不完整。
默认情况下的完整扫描间隔是 3600 秒。用户可以根据实际情况调整完整扫描间隔，以满足不同的需求和场景。需要注意的是，完整扫描间隔只是系统进行完整扫描的时间间隔，系统会在必要时进行增量扫描和同步，以确保文件的一致性和完整性。
## 文件拉取顺序
文件拉取顺序指的是同步文件时的拉取顺序，可以根据不同的需求和场景选择不同的拉取顺序。具体的文件拉取顺序包括以下几种：
a. 随机顺序：随机选择要同步的文件，适用于同步文件数量较少的情况。
b. 字母顺序：按照文件名的字母顺序进行同步，适用于同步大量文件时，希望按照字母顺序进行同步的情况。
c. 小文件顺序：先同步小文件，适用于同步大量文件时，希望先同步小文件，减少同步时间的情况。
d. 大文件顺序：先同步大文件，适用于同步大量文件时，希望先同步大文件，以便更快地完成同步的情况。
e. 旧文件顺序：按照文件的修改时间进行同步，先同步旧文件，适用于同步频繁修改的文件夹的情况。
f. 新文件顺序：按照文件的修改时间进行同步，先同步新文件，适用于同步不经常修改的文件夹的情况。
需要注意的是，不同的文件拉取顺序适用于不同的场景和用途。用户可以根据自己的需求选择适当的文件拉取顺序，以实现更加灵活和高效的文件同步。
## 忽略文件权限
忽略文件权限是一个高级选项，用于控制是否在文件同步时忽略文件的权限信息。如果用户选择了忽略文件权限选项，当同步文件夹时，系统会忽略文件的权限信息，即不考虑文件的所有者、组和权限等信息，直接同步文件内容。
忽略文件权限选项通常用于跨平台同步，例如在 Linux 和 Windows 之间同步文件时，由于文件权限的差异，可能会导致同步失败。此时，选择忽略文件权限选项可以避免这种问题，确保文件内容的同步。
需要注意的是，如果选择了忽略文件权限选项，可能会导致文件的安全性和完整性受到影响。建议在必要的情况下才使用该选项，并确保文件的安全性和完整性得到保障。
## 扩展属性
同步扩展属性和发送扩展属性是两种不同的选项，用于控制是否在文件同步时保留文件的扩展属性信息。它们的含义如下：
a. 同步扩展属性：如果选择了同步扩展属性选项，当用户同步文件夹时，系统会尝试将文件的扩展属性信息（例如文件的创建时间、修改时间、元数据等）与远程设备同步。这意味着，如果远程设备没有相应的扩展属性信息，系统会尝试将本地设备的扩展属性信息应用到文件上。这种方式可以确保文件在不同设备之间的扩展属性信息相同。
b. 发送扩展属性：如果选择了发送扩展属性选项，当用户同步文件夹时，系统只会将文件发送到远程设备，并尝试将本地设备的扩展属性信息应用到文件上。这意味着，如果远程设备没有相应的扩展属性信息，文件的扩展属性信息可能会发生变化。
需要注意的是，这些扩展属性选项只在文件同步时起作用，并不会影响文件本身的属性信息。如果用户需要更精细地控制文件的属性信息，可以使用操作系统提供的相应工具进行设置。

## 地址列表
地址列表是设备的网络地址列表，用于指定设备的可访问地址。在编辑设备默认值时，用户可以添加设备的地址列表，以便在添加新设备时自动应用这些地址。此处可以输入以半角逗号分隔的（" tcp://ip:port ", " tcp://host:port "）设备地址列表，或者输入“ dynamic ”以自动发现设备地址。
## 压缩
对文件在传输过程中的压缩情况做选择，分为仅元数据、所有数据和关闭 3 个选项。压缩选项会影响文件传输的速度和质量，用户可以根据自己的需求和网络环境选择适当的压缩选项。
a. 仅元数据：该选项表示在文件传输过程中只压缩文件的元数据（如文件名、大小、时间戳等），而不压缩文件内容。这样可以减少传输过程中的网络带宽和 CPU 占用率，但文件传输速度可能会变慢。
b. 所有数据：该选项表示在文件传输过程中压缩所有数据，包括文件的元数据和内容。这样可以更快地传输文件，但会占用更多的网络带宽和 CPU 资源。
c. 关闭：该选项表示在文件传输过程中不进行压缩，直接传输文件内容。这样可以保持文件的原始格式和完整性，但会占用更多的网络带宽和 CPU 资源。
## 设备速率限制
下载速率限制：用于限制从该设备下载文件的速率，以避免网络拥塞和影响其他应用程序的网络使用。
上传速率限制：用于限制向该设备上传文件的速率，以避免网络拥塞和影响其他应用程序的网络使用。
## 不可信的
该选项用于标记该设备为不可信任的设备，勾选后与此设备共享的所有文件夹都必须有密码保护，这样所有发送的数据在没有密码的情况下是不可读的，以确保文件的安全性和完整性。

GUI 监听地址：用于指定 SyncService GUI 的监听地址，即 SyncService GUI 的访问地址。默认情况下，SyncService GUI 监听在本地 IP 地址的端口号 8384 上。如果需要从外部访问 SyncService GUI，可以将 GUI 监听地址设置为公共 IP 地址或域名。
图形管理界面用户名和密码：用于设置 SyncService GUI 的登录用户名和密码，以便只有授权用户才能访问 GUI 。用户可以在该选项中设置用户名和密码，以保护 GUI 的安全性。
使用加密连接到图形管理页面：用于指定是否在访问 SyncService GUI 时使用加密连接。如果选择了该选项，则 SyncService GUI 会使用 HTTPS 协议进行加密连接，以保护数据的安全性和隐私性。
是否启动浏览器：用于指定是否在启动 SyncService 时自动打开浏览器访问 SyncService GUI 。如果选择了该选项，则启动 SyncService 时会自动打开浏览器，并访问 SyncService GUI 的地址。
GUI 主题：SyncService 的图形用户界面（GUI）提供了多种主题，以便用户可以根据个人喜好和需求选择合适的主题。

协议监听地址：该选项用于指定 SyncService 监听连接的地址和端口号。

全局发现服务器：该选项用于指定全局发现服务器的地址，以便 SyncService 可以在互联网上搜索其他设备进行连接。如果需要在互联网上连接其他设备，用户可以添加全局发现服务器的地址。需要注意的是，这些选项可以根据实际情况进行设置，以保障 SyncService 的连接和性能。特别是下载速率限制、上传速率限制、启用 NAT 穿透和启用中继等选项，可以帮助用户控制网络带宽和保障连接的稳定性和速度。

### 3.1.2 操作-显示ID
1696829562143-1bc9ec36-c308-4f01-a00c-94e36e77b472.png

设备标识是指每个设备的唯一标识符。设备标识是由 SyncService 自动生成的，用于标识每个连接到同步网络的设备。当用户与其他设备建立连接时， SyncService 会自动分配一个唯一的设备标识符，并将其显示在显示 ID 按钮下。
设备标识在 SyncService 中是非常重要的，因为它们用于建立设备之间的安全连接和同步通信。通过检查设备标识，用户可以确定正在连接的设备是否是你想要连接的设备，从而避免安全问题和数据泄露。

### 3.1.3 操作-关闭
关闭操作是指停止 SyncService 进程或关闭 SyncService 界面。关闭 SyncService 将停止所有正在进行的同步操作，并且 SyncService 不再监听新的同步请求或连接请求。
### 关闭 SyncService 的作用包括：
停止所有同步操作：关闭 SyncService 将停止所有正在进行的同步操作，包括上传和下载文件。这有助于避免同步冲突和数据丢失。
节省系统资源：当 SyncService 运行时，它会占用一定的系统资源，包括 CPU、内存和网络带宽。关闭 SyncService 可以释放这些资源，让系统更加流畅。
增强安全性：关闭 SyncService 可以避免其他人通过 SyncService 访问共享文件夹和设备，从而增强数据安全性。
需要注意的是，关闭 SyncService 将停止所有同步操作，包括正在进行的和尚未完成的同步任务。如果在关闭 SyncService 之前有未完成的同步任务，建议在关闭之前先等待同步任务完成，以避免数据丢失和同步冲突。

### 3.1.4 操作-重启
重启操作是指关闭 SyncService 进程并重新启动它。重启 SyncService 将重新加载配置文件和插件，并重新建立与其他设备的连接。重启 SyncService 通常用于解决一些常见的问题，如插件加载失败、网络连接问题等。
### 重启 SyncService 的作用包括：
重新加载配置文件：重启 SyncService 将重新加载配置文件，包括共享文件夹、设备列表和网络设置等，这有助于解决一些配置问题。
重新建立连接：重启 SyncService 将重新建立与其他设备的连接，包括连接失败的设备，这有助于解决一些网络连接问题。
解决插件问题：重启 SyncService 可以解决一些插件加载失败或崩溃的问题。
需要注意的是，重启 SyncService 将停止所有正在进行的同步操作，包括正在进行的和尚未完成的同步任务。因此，在重启 SyncService 之前，建议先等待同步任务完成，以避免数据丢失和同步冲突。
