# Windows

## Windows

### 对于 windows 平台，目前支持如下操作系统：
## windows server 2016

对于 windows 10 等非服务器系统，RDS 可以运行但不保证其运行稳定性。
## 1 安装
将最新的 RDSInstallers 安装包，如 Base_RDSInstaller_4.3.0.0.exe（for Windows）安装在目标主机上，直接安装即可（建议安装在根目录下，同时保证计算机名称为英文）。

## 2 更新
用户可以通过 Roboshop 来更新 Core 和 RDS 。（需在 Robod 软件运行时）。
1）在目标服务器上，运行 RDS（安装完成后点击桌面上的 RDS 蓝色图标，双击启动）。在同一局域网的 PC 上打开 Roboshop 软件，点击界面右上角的【添加设备】：
1657534867335-6cfd8792-332c-441b-9478-7c10d13e6aa6.png

2）输入目标服务器的 IP（如127.0.0.1），点击【辅助程序】：
1657534897665-f59bada3-8868-4970-b76a-86b9f51b8953.png

3）选择【升级/备份】，将需要升级的 Core 和 RDS 的安装包拖拽到指定位置（也可通过 加载文件 按钮选择 RDS 安装包），点击【开始升级】，如下图：
1657534908192-651777b0-b33f-4be3-8262-3ee939fd4407.png

4）升级成功后，需要等待不超过 30s。待服务器上 RDS 命令行重启，表明更新成功。如果当前操作的 PC 不是调度服务器，可尝试用浏览器登录 RDS（在浏览器中输入 localhost:8080 ），若能够出现登录页面，也表示更新成功。

## 3 授权
在 上一节，使用 Roboshop 更新 Core 和 RDS 成功后，切换到【版本信息】页面。或者在调度服务器中点击高级设置，点击【版本信息】，此时页面右侧 Core 的状态将显示【未激活】，激活该条表示获得标准版的 Core 和 rds 的授权。另外， rds_custom 和 rds_operator 分别表示低代码和手持端的授权，也可以一并激活。您需要复制【机器码】，将【机器码】提交我司项目经理，索要授权文件。后点击【离线激活】按钮，使用授权文件激活 Core 的对应权限。（ 注：激活的过程中，Core 将会重启，属正常现象 ）。

Core 激活后会出现【已激活】提示。

## 注意：
在目标服务器上，只需安装一次运行环境即可。后续升级 Core 整包时，可直接使用 Roboshop 进行。
RDS 后台管理界面需要用户名及密码登录， 若浏览器无法正常访问目标地址，则说明 RDS 启动不正常。

### 4 设置 Base 开机不自启（服务器慎用！）
## 【编写目的】
部分用户，是在个人电脑（PC）上通过 Base 工具安装的 RDS 。
默认状态下， RDS 是开机自启的，一定程度上影响了这些用户的工作。
本小节的内容，就是为了解决此问题。

## 【注意事项】
### 此小节的内容，仅适用于 Windows 操作系统！
此小节的内容，仅适用于在个人电脑（PC）上，通过 Base 工具安装 RDS 的情况。 服务器慎用本小节的操作！！！
Base 其实就是 Robod 或者 RobodPro ，本小节统一用 RobodPro 进行说明 。

## 【操作步骤】
1660014017848-daa964f6-36d9-4d3c-bbb9-2c8bb0ec82be.png 

1660014045912-5da46154-abe2-4466-932e-1e76937fb4fe.png 

① 找到配置文件所在目录 C:\RDS\data\robod\appInfo\setting 。
② 编辑配置文件 Config.ini 。
③ 设置项 autoStartApp 表示是否开机自启动 RobodPro ，初始值为 true ，即系统启动之后，就会自动运行 RobodPro 。 如果我们不需要开机自动运行 RobodPro ，则只需要将 autoStartApp 的值改为 false 。
④ 找到 RobodPro 的启动文件 RobodPro.exe 所在的目录 C:\RDS\data\robod 。
⑤ 右击 RobodPro.exe ，会弹出一个操作菜单。
⑥ 在弹出的菜单中，选择 以管理员身份运行 。
不通过管理员身份运行 RobodPro.exe ，修改后的配置是不会生效的！
然后可以重启操作系统进行验证。
