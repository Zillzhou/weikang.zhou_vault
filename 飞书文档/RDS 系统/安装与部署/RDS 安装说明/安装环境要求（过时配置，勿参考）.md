# 安装环境要求（过时配置，勿参考）

## 安装环境要求（过时配置，勿参考）
## 运行平台：
提供的典型服务器：戴尔 T150服务器。
中配建议：英特尔至强银牌 4112，32GB RAM，1TB SSD，4核8线程。
低配建议：英特尔至强 E-2224，16GB RAM，1TB SSD，4核4线程。

Windows 系统要求（不推荐，未来将停止维护 Windows 版本）：
## Windows Server 2016
## Windows Server 2019
Windows 10 及以上（其他 Windows 系统可能存在兼容问题）
## Windows Server 2022

## Linux 系统要求：
Ubuntu 18.04。
Ubuntu 20.04。

## 注意：
1.对于 windows 7 系统的服务器或个人PC，RDS可以运行但不保证其运行稳定性； 使用Base 4.3.5 安装软件 ，后安装 低版本数据库mariadb-10.3.31-winx64 ，
### 2.注意，对于所有的操作系统来说，防火墙策略一定不能过滤所有我方系统使用到的端口号（端口具体参考： 常用端口 ）；
### 3.操作系统设置的登录账号，必须具备管理员账号的所有权限，不能设置访问限制、文件读取限制等，否则我方系统不能进行正常使用；

## 云服务器要求
### 云服务器内的操作系统要求如上文所述，其他相关要求如下:
## 1.服务器内存 ≥ 16G。依据项目机器人数量及业务复杂度而定。
### 2.服务器核心数 ≥ 4核。
### 3.云服务器的端口需支持对外开放，这些端口是服务器内的调度系统使用，需要开放给厂区机器人连接，调度系统相关端口清单见 端口文档 。
### 4.厂区内机器人和云服务器网络（ping）延迟小于60ms。
## 软件环境
## OpenJDK 11
OpenJDK 11是开源免费可商用的 Java 运行环境。

### mariadb-server (10.6.8)
开源数据库，需设置 root 用户及密码，并配置数据库表忽略大小写。

## Core
依赖的底层调度算法系统。

## Robod
 Robod 为 RDS 运行的守护进程，提供 RDS 和 Core 的升级、激活和日志提取等功能 。
