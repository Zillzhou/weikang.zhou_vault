# RDS 目录结构引起 RDS 服务无法启动

### RDS 目录结构引起 RDS 服务无法启动
1666338881691-0be38163-478e-495b-85fc-91d3e8f9ac5c.png

RDS 的安装目录下的 rds 文件夹下存放的是 rds 服务的项目文件。其中：
app 文件夹下存放的是 rds 项目的 jar 包、项目的配置文件 application.yml 、项目的启动文件。app文件夹下缺失这些文件 rds 服务不能启动
1666339698087-db9066ab-106d-47e4-b701-ffb2f5f55b81.png

config 文件夹下存放的是项目第一次启动时需要创建的数据库表的 sql 文件，及天风任务块文件和机器人执行动作文件，若第一次安装 rds 是缺少 sql 文件， rds 的服务无法启动，其余的不会影响服务的启动。其中，天风任务块文件每次启动都会生成一份。机器人执行动作文件不会生成，缺失会造成机器人脚本名为空。

1666340059801-b974213f-0d32-4f6c-91e6-56bde46176a7.png

script 目录下存放的是脚本文件及 RDS 配置菜单的配置文件，首次启动此文件不存在，后续添加脚本、配置保存后会在此文件夹中生成 *.js 文件、 application-biz.yml 的配置文件。

static 目录下存放的 RDS 页面，缺少此文件下的文件，通过 url 地址访问 RDS 时出现 登录状态失效，需重新登录 字样
